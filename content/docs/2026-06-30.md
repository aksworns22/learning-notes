## 멀티 헤드 어텐션

<p>지난 번 단일 어텐션 헤드에서 한 발 더 나아가, 같은 입력을 <strong>여러 관점</strong>으로 동시에 바라보는 멀티 헤드 어텐션을 구현했다. 핵심 아이디어는 고정된 토큰 임베딩을 헤드 수(<code>n_head</code>)만큼 잘라, 각 조각을 별도의 <code>W_q</code>, <code>W_k</code>, <code>W_v</code>로 통과시켜 서로 다른 부분 공간에서 관계를 계산하는 것이다.</p>

<ul>
  <li>입력 <code>(B, T, C)</code>를 <code>W_q/W_k/W_v</code>로 <code>d_model</code> 차원에 투영한 뒤 <code>(B, T, n_head, C/n_head)</code>로 쪼개고, <code>transpose</code>해서 <code>(B, n_head, T, C/n_head)</code> 형태로 만든다.</li>
  <li>각 헤드별로 스케일드 닷프로덕트 어텐션을 독립적으로 수행한다.</li>
  <li>헤드별 결과를 다시 이어 붙여(<code>(B, T, C)</code>) 마지막 선형 층(<code>concat_linear</code>)으로 섞어준다.</li>
</ul>

<p>한 헤드는 문법 관계에, 다른 헤드는 의미 관계에 집중하는 식으로 여러 관점의 정보를 한 번에 담을 수 있다. 어텐션을 적용해도 텐서의 전체 형태 <code>(1, 5, 16)</code>는 그대로 유지된다 — 바뀌는 것은 각 토큰 임베딩이 담는 정보다.</p>

<details>
<summary>멀티 헤드 어텐션 구현</summary>
<pre><code>class MultiHeadAttention(nn.Module):
    def __init__(self, token_embed_dim, d_model, n_head, is_causal=False):
        super().__init__()
        self.n_head = n_head
        self.is_causal = is_causal
        self.weight_q = nn.Linear(token_embed_dim, d_model)
        self.weight_k = nn.Linear(token_embed_dim, d_model)
        self.weight_v = nn.Linear(token_embed_dim, d_model)
        self.concat_linear = nn.Linear(d_model, d_model)

    def forward(self, queries, keys, values):
        B, T, C = queries.size()
        # (B, T, C) -> (B, T, n_head, C/n_head) -> (B, n_head, T, C/n_head)
        queries = self.weight_q(queries).view(B, T, self.n_head, C // self.n_head).transpose(1, 2)
        keys = self.weight_k(keys).view(B, T, self.n_head, C // self.n_head).transpose(1, 2)
        values = self.weight_v(values).view(B, T, self.n_head, C // self.n_head).transpose(1, 2)
        attention = compute_attention(queries, keys, values, is_causal=self.is_causal)
        output = attention.transpose(1, 2).contiguous().view(B, T, C)
        return self.concat_linear(output)

n_head = 4
multi_head_attention = MultiHeadAttention(embedding_dim, embedding_dim, n_head)
out = multi_head_attention(input_embeddings, input_embeddings, input_embeddings)
# 멀티 헤드 어텐션 적용 후 형태: (1, 5, 16)
</code></pre>
</details>

## 정규화와 피드 포워드 층

<p>딥러닝 모델은 입력 데이터의 분포가 서로 다르면 학습이 잘 되지 않기 때문에 정규화가 중요하다. 정규화는 <code>norm_x = (x - 평균) / 표준편차</code> 형태로, 평균과 표준편차를 구할 데이터를 어떻게 묶느냐에 따라 나뉜다.</p>

<ul>
  <li><strong>배치 정규화</strong>: 주로 이미지에서 사용.</li>
  <li><strong>층 정규화(Layer Norm)</strong>: 자연어 처리에서 사용. 각 토큰의 임베딩 벡터 <em>안에서</em> 값들의 평균·분산을 정리해 학습을 안정화한다.</li>
</ul>

<p><strong>피드 포워드 층</strong>은 데이터의 특징을 학습하는 완전 연결 층(Fully Connected Layer)이다. 멀티 헤드 어텐션이 단어 <em>사이</em>의 관계를 파악하는 일이라면, 피드 포워드 층은 입력 텍스트 전체를 이해하는 역할을 담당한다.</p>

<p>인코더 층은 아래 그림처럼 <strong>층 정규화 → 멀티 헤드 어텐션 → 잔차 연결 → 층 정규화 → 피드 포워드 → 잔차 연결</strong> 구조를 <code>N_e</code>번 반복한다. 정규화를 어텐션·피드포워드 <em>앞에</em> 두는 방식을 Pre-Layer Norm이라고 한다.</p>

<pre><code>norm = nn.LayerNorm(embedding_dim)

class PreLayerNormFeedForward(nn.Module):
    def __init__(self, d_model, dim_feedforward, dropout):
        super().__init__()
        self.norm = nn.LayerNorm(d_model)
        self.linear1 = nn.Linear(d_model, dim_feedforward)
        self.linear2 = nn.Linear(dim_feedforward, d_model)
        self.dropout1 = nn.Dropout(dropout)
        self.dropout2 = nn.Dropout(dropout)
        self.activation = nn.GELU()

    def forward(self, src):
        x = self.norm(src)
        x = x + self.linear2(self.dropout1(self.activation(self.linear1(x))))
        x = self.dropout2(x)
        return x

class TransformerEncoderLayer(nn.Module):
    def __init__(self, d_model, nhead, dim_feedforward, dropout):
        super().__init__()
        self.attention = MultiHeadAttention(d_model, d_model, nhead)
        self.norm1 = nn.LayerNorm(d_model)
        self.feedforward = PreLayerNormFeedForward(d_model, dim_feedforward, dropout)

    def forward(self, src):
        norm_x = self.norm1(src)
        attention_output = self.attention(norm_x, norm_x, norm_x)
        x = src + attention_output          # 잔차 연결
        return self.feedforward(x)

## 마스크 어텐션 (인과적 어텐션)

<p>트랜스포머 모델은 앞에서 생성한 토큰을 기반으로 다음 토큰을 생성한다. 이렇게 순차적으로 생성해야 하는 특징을 <strong>인과적(causal)</strong> 또는 <strong>자기-회귀적(auto-regressive)</strong>이라고 한다. 디코더는 학습/생성 시 미래 토큰을 미리 봐서는 안 되므로, 현재 위치보다 뒤에 오는 토큰의 어텐션 스코어를 가린다.</p>

<p>구현은 간단하다. 어텐션 스코어 행렬에서 대각선 위쪽(미래 토큰) 자리를 <code>-inf</code>로 채운다. <code>-inf</code>에 softmax를 취하면 0이 되므로, 미래 토큰은 가중치 0이 되어 반영되지 않는다.</p>

<img src="/learning-notes/images/2026-06-30-2.png" alt="마스크 연산: 어텐션 스코어 행렬의 대각선 위쪽을 -inf로 채워 미래 토큰을 가린다">

<details>
<summary>인과적 마스크가 적용된 어텐션 구현</summary>
<pre><code>def compute_attention(queries, keys, values, is_causal=False):
    dim_k = queries.size(-1)
    scores = queries @ keys.transpose(-2, -1) / sqrt(dim_k)
    if is_causal:
        query_length = queries.size(-2)
        key_length = keys.size(-2)
        # 하삼각(tril)만 True로 두고, 나머지(미래 토큰) 자리는 -inf로 채움
        temp_mask = torch.ones(query_length, key_length, dtype=torch.bool).tril(diagonal=0)
        scores = scores.masked_fill(temp_mask == False, float('-inf'))
    weights = F.softmax(scores, dim=-1)   # -inf -> 0
    return weights @ values
</code></pre>
</details>

## 크로스 어텐션과 디코더 층

<p>인코더와 디코더의 차이는 크게 두 가지다 — <strong>마스크 어텐션</strong>과 <strong>크로스 어텐션</strong>. 크로스 어텐션은 인코더의 결과를 디코더에서 활용하는 연산이다. 예를 들어 영어를 한국어로 번역할 때, 인코더가 영어 문장을 처리한 결과(<code>encoder_output</code>)를 한국어를 생성하는 디코더가 받아 참고한다.</p>

<ul>
  <li><strong>셀프 어텐션(마스크 적용)</strong>: 디코더가 지금까지 생성한 토큰들끼리의 관계를 계산(<code>x, x, x</code>). 미래 토큰은 마스킹.</li>
  <li><strong>크로스 어텐션</strong>: 쿼리는 디코더(<code>x</code>)에서, 키·값은 인코더 출력(<code>encoder_output</code>)에서 가져와 입력 문맥을 끌어온다.</li>
</ul>

<div class="mermaid">
graph TD
  enc["인코더 출력<br>(영어 문장 처리 결과)"] --> cross
  tgt["디코더 입력 (tgt)"] --> sa["마스크 셀프 어텐션<br>self_attention(x, x, x)"]
  sa --> add1["잔차 연결 + 층 정규화"]
  add1 --> cross["크로스 어텐션<br>cross_attention(x, enc, enc)"]
  cross --> add2["잔차 연결"]
  add2 --> ff["피드 포워드 층"]
  ff --> out["디코더 층 출력"]
</div>

<details>
<summary>디코더 층 구현</summary>
<pre><code>class TransformerDecoderLayer(nn.Module):
    def __init__(self, d_model, nhead, dim_feedforward=2048, dropout=0.1):
        super().__init__()
        self.self_attention = MultiHeadAttention(d_model, d_model, nhead, is_causal=True)
        self.cross_attention = MultiHeadAttention(d_model, d_model, nhead)
        self.norm1 = nn.LayerNorm(d_model)
        self.norm2 = nn.LayerNorm(d_model)
        self.dropout1 = nn.Dropout(dropout)
        self.dropout2 = nn.Dropout(dropout)
        self.feedforward = PreLayerNormFeedForward(d_model, dim_feedforward, dropout)

    def forward(self, tgt, encoder_output, is_causal=True):
        x = self.norm1(tgt)
        # 1) 마스크 셀프 어텐션
        x = x + self.dropout1(self.self_attention(x, x, x, is_causal=is_causal))
        x = self.norm2(x)
        # 2) 크로스 어텐션: 쿼리는 디코더, 키/값은 인코더 출력
        x = x + self.dropout2(self.cross_attention(x, encoder_output, encoder_output))
        return self.feedforward(x)
</code></pre>
</details>

## BERT, GPT, T5 — 트랜스포머 활용 아키텍처

<p>같은 트랜스포머 블록을 어떻게 조합하느냐에 따라 모델 계열이 갈린다.</p>

<img src="/learning-notes/images/2026-06-30-3.png" alt="인코더/디코더/인코더-디코더 모델 그룹별 대표 모델과 장단점 표">

| 모델 그룹 | 대표 모델 | 특징 |
| --- | --- | --- |
| 인코더 | 구글 BERT | 양방향 문맥 이해에 강해 자연어 이해(분류 등)에 적합. 생성 작업에는 부적합. |
| 디코더 | OpenAI GPT | 자기-회귀적 생성에 강함. 단방향이라 입력 전체를 양방향으로 보지는 못함. |
| 인코더-디코더 | 메타 BART, 구글 T5 | 입력 이해(인코더) + 출력 생성(디코더)을 모두 갖춰 번역·요약에 적합. 구조가 무겁고 학습/추론 비용이 큼. |

## 주요 사전 학습 메커니즘

<p>모델 계열의 차이는 <strong>사전 학습 방식</strong>의 차이에서 비롯된다.</p>

<ul>
  <li><strong>인과적 언어 모델링 (Causal LM)</strong> — 디코더 모델 / GPT 계열. 문장의 시작부터 끝까지 순차적으로 다음 단어를 예측한다. <strong>단방향</strong>.</li>
  <li><strong>마스크 언어 모델링 (Masked LM)</strong> — 인코더 모델 / BERT 계열. 입력 단어의 일부를 <code>[MASK]</code>로 가리고 그 단어를 맞춘다. 앞뒤를 모두 보는 <strong>양방향</strong>.</li>
</ul>

<img src="/learning-notes/images/2026-06-30-4.png" alt="MLM 학습 방식: 입력의 일부를 [MASK]로 가리고 BERT가 가려진 원래 단어를 예측">

<p>그림의 MLM 예시처럼, "먹고"를 <code>[MASK]</code>로 가린 뒤 BERT가 후보 단어(먹고 15%, 걷고 5% …) 중 원래 단어를 맞추도록 학습한다. 이렇게 양방향 문맥을 활용하는 점이 단방향으로 다음 토큰만 예측하는 GPT와 대비된다.</p>
