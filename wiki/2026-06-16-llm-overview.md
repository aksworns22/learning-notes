# LLM 지도: 딥러닝과 언어 모델링

## 핵심 개념

**임베딩**: 데이터의 의미와 특징을 포착해 숫자로 표현한 것

**언어 모델링**: 모델이 입력받은 텍스트의 다음 단어를 예측해 텍스트를 생성하는 방식
- 언어 모델(language model): 다음 단어를 예측하는 방식으로 훈련한 모델

## 딥러닝 vs. 머신러닝

딥러닝이 머신러닝과 가장 큰 차이를 보이는 지점: **특징(feature)을 누가 뽑는가**

| | 특징 추출 주체 |
|--|--|
| 기존 머신러닝 | 연구자 / 개발자 |
| 딥러닝 | 모델이 스스로 |

## LLM 역사의 세 가지 전환점

| 연도 | 사건 |
|------|------|
| 2013 | word2vec — 단어를 벡터로 표현, 의미 유사도 포착 |
| 2017 | Transformer architecture 등장 |
| 2018 | GPT-1 — 언어 모델링으로 사전 학습 후 fine-tuning이 효과적임을 증명 |

## 전이 학습 (Transfer Learning)

하나의 문제를 해결하는 과정에서 얻은 지식을 다른 문제에 활용하는 방식

- **사전 학습(pre-training)**: 대량의 데이터로 모델을 먼저 학습
- **미세 조정(fine-tuning)**: 특정 다운스트림 과제를 위해 추가 학습

NLP에서 돌파구: 2018년, 다음 단어를 예측하는 언어 모델링 방식으로 사전 학습 시 적은 레이블 데이터로도 기존 지도 학습을 뛰어넘는다는 것을 발견 → GPT-1

## RNN의 한계

**시퀀스**: 작은 단위(단어)의 데이터가 연결되고, 길이가 다양한 데이터 형태

RNN은 하나의 잠재 상태(hidden state)에 지금까지의 입력 텍스트 맥락을 압축한다.
- 효율적이지만, 입력이 길어질수록 먼저 입력한 단어의 의미가 희석됨
- → 이 한계를 극복하기 위해 Transformer(Attention 메커니즘)가 등장

## 언어 모델이 ChatGPT가 되기까지

핵심 개념: **트랜스포머 + 정렬(alignment)**

- Transformer: 시퀀스 전체를 병렬로 처리, 장거리 의존성 포착 가능
- Alignment(정렬): 모델 출력이 인간의 의도에 맞도록 조정하는 과정 (RLHF 등)
